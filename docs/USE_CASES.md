# Use Cases

Use this page when you want to choose the right Codex Maintainer Kit command.

## 1. Run a maintenance audit

Use `audit` when you want a maintenance audit before asking Codex or another agent to change a repository.

Good output: score, missing maintainer files, test/documentation gaps, and prioritized next actions.

## 2. Create a maintainer brief

Use `brief` when you want one context file that explains the repository to Codex before a maintenance session.

Good output: repository readiness checklist, suggested Codex prompt, human review rule, and task queue.

## 3. Turn gaps into tasks

Use `tasks` when you want Markdown, JSON, or GitHub issue drafts from the repository readiness gaps.

Good output: actionable task titles, labels, verification commands, and maintainer checklists.

## 4. Review AI-assisted changes

Use `review` after local changes exist and a human maintainer needs a structured review report.

Good output: changed files, risk notes, verification commands, and merge guidance.

## 5. Pair with GitHub Actions

Use the CLI locally for hands-on inspection. Use [Codex Maintainer Action](https://github.com/goonobu-dot/codex-maintainer-action) when you want scheduled or pull-request based artifact generation.
