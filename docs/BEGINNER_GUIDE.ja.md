# Codex Maintainer Kit やさしい解説

このページは、Codex Maintainer Kit を中学生でも分かるように説明するページです。

## このプロジェクトは何か

Codex Maintainer Kit は、GitHubの公開プロジェクトを点検するための道具です。

プログラムを公開したあとも、次のような手入れが必要になります。

- READMEを分かりやすくする
- ライセンスを用意する
- テストを増やす
- セキュリティの説明を書く
- IssueやPull Requestを整理する

この道具は、プロジェクトを見て「次に何を直すとよいか」を整理します。

## 何のために役立つか

- OSSの健康診断をする
- Codexに頼む作業リストを作る
- 人間がレビューしやすい形にまとめる
- 足りないREADME、テスト、CI、SECURITYなどを見つける

## 作られる主なファイル

- `OSS_MAINTENANCE_AUDIT.md`: プロジェクトの点検結果
- `MAINTAINER_BRIEF.md`: メンテナー向けのまとめ
- `CODEX_TASKS.md`: Codexに頼める作業リスト
- `codex-tasks.json`: 機械が読みやすいタスク情報
- `CODEX_REVIEW.md`: 変更内容を人間が確認するためのメモ

## たとえ話

部活の道具箱を点検するとします。

ボールはあるか、空気入れはあるか、ルール表はあるか、壊れているものはないかを確認します。

Codex Maintainer Kit は、GitHubプロジェクトに対して同じような点検をします。

## 注意点

この道具は、AIに勝手に全部直させるものではありません。  
作業リストや点検表を作り、最後に人間が確認します。

つまり、AIに丸投げするための道具ではなく、AIを安全に使うための準備道具です。
